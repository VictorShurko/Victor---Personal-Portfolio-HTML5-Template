Getting Started

Installing Template

After you download your website template from ThemeForest and make all of your edits you're going to need to upload the files to your web server. You will need an FTP client such as Dreamweaver or FileZilla. In your FTP client add your server details (get this from your web host provider) and then you can connect to your server and upload your files.

If you haven't FTP Client you can download it from Filezilla's official website.

Updates

When an update is available in the future you will need to re-download the package from ThemeForest. While logged into your account at ThemeForest hover over your name in the top right of the screen and click Downloads. From here Download > All files button next to Victor item and save the package on your computer.

Template Structures

HTML Structure

This template is a flexible layout. The main contents are inside the 'body' tag divided into differt section (i.e. header, about, service, ..., footer etc.).

CSS Structure

I'm using 3 CSS files, one empty file 'custom.css'(for your code) and several vendor CSS files in this template.

JavaScript Structure

All call actions are run in main.js

The slider.js file contains the settings for the Swiper slider.

The email.js file contains the code that is used to validate the contact form, display messages and send the contact form to the server.

Contact Form

Contact form works only in hosting environment. If you want to use it, open "dist/php/contact.php" file and change the main email address in a variable - $email_it_to (line-13) to yours.
Important! Some hosting providers don't support PHP Mail Function automatically. So if you don't receive any message after changing mail address to yours, then contact with your hosting provider and they'll enable PHP Mail function.

Code Editor

In order to edit the file, you need to have Code Editor program installed on your system. Just open files in code writer and edit them. Recommended programs are Visual Studio Code, Adobe Dreamweaver

RTL

If you need a site with RTL then make all changes and file manipulations in the 'dist(RTL)' folder.

Site Title and Meta Tags

To change your Site title and Meta tags open the file 'index.html' in your editor and go to the location by following screenshot which are given below.

Favicon

Upload your favicon to the 'images/favicon' folder.
There are a few different sizes you should create so it looks great on retina screens and different platforms.
For your convenience, you can use a special service for generating favicons, such as RealFaviconGenerator.
When uploading your files to the favicon folder, make sure that the file names have not changed. The file name should be the same.

Logotype

Upload your logotype to the 'images' folder. The logo must be in svg format.
Make sure that the file name does not change.The file name should be the same.

Animated Headline

Delete words or add new words in the animated headline located in the header.

Changing Images

Images are recommended to use in WebP format.

WebP is a modern image format that provides superior lossless and lossy compression for images on the web. Using WebP, you can create smaller, richer images that make the web faster.

If you are uploading your image in a different format or with a different name than the one in the image folder, then please change the image name in the 'index.html' file code.

CV (Resume)

Upload your CV file in PDF format to your dist or dist(RTL) folder. The file name must be 'resume.pdf'.

If you don't need a CV file, comment out or remove some of the code in the 'index.html' file.

Social Profile

Pul your social profile link here.

Skills

Change the number in the 'data-progress' attribute to your number from 0 to 100.

The data-progress number indicates how much the skill bar in the skill block will fill up.

Portfolio

Add a category to the element in the 'data-category' attribute. This category should be in the list of categories in the block with id="filter__categories".

Blog

The 'blog' folder contains the article file. After creating the article, do not forget to add a link to the article on the main page of the site, in the 'blog' section.

RewriteEngine on
ErrorDocument 404 /404.html
